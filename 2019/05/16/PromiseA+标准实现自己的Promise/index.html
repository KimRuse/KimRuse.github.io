Promise A/+ 标准实现自己的 Promise

Promise A/+规范 文档地址

规范中的术语

  1. “promise” 是拥有符合规范的then方法的一个对象或函数
  2. “thenable” 一个定义了then方法的对象或函数
  3. “value” 任意的合法JS值
  4. “reason” 一个指示promise为什么被拒绝的值
  5. “exception” 一个通过throw语句抛出的值

Promise状态

一个Promise必须是pending、fulfilled、rejected三种状态之一

  1. pending状态时，可转换为 fulfilled 或 rejected 状态
  2. fulfilled状态时，不能转化其他状态 且 必须包含value，value不能改变

  3. rejected状态时，不能转化其他状态 且 必须包含reason，reason不能改变

    ( 不能改变是指，通过===判定相等 即不变，非深层不变 )

接下来实现resolve 和 reject 处理变更后的状态和值，及可能的回调

then方法

一个promise的then方法接受两个参数

  1. onFulfilled 和 onRejected 为可选参数，当不是function时，必须忽略；pending状态改变时 promise2 的value或reason 必须等于promise1
  2. onFulfilled 和 onRejected 只能被调用一次，并且在promise决议之后调用，使用promise的value或reason作为其第一参数
  3. onFulfilled 和 onRejected 必须在当前执行上下文栈清空后，被异步调用，根据规范提示我们可以用一个”macro-task”实现，如setTimeout，或一个”micro-task”
  4. onFulfilled 和 onRejected 返回一个值x, 运行[[Resolve]] (promise,x)，抛出e时，promise2必须rejected(e)
