Javascript 中的类型转换

目前JS中有8种类型，Undefined、Null、Boolean、Number、String、Symbol、BigInt、Object

以下皆为Number(), String(), Boolean() 函数转换说明，隐式转换一样 此如+’12a’会调Number()返回NaN

原始值 -> 转换的类型

原始值 -> 布尔类型Boolean() 只有6种值会被转换成false（false, undefined, null, +0, -0, NaN, “”），其余转true

一般使用parseInt和parseFloat，尽可能多解析数值字符，并忽略后面的内容，空字符串和第一个非数值字符返回NaN；

原始值 -> 对象类型 通过String()、Number()、Boolean()构筑函数，转化为包装对象

null 和 undefined 例外， 用在期望是一个对象的地方会造成（TypeError）异常，比如 undefined.a 调用a属性

对象 -> 转换的类型

对象-> 布尔类型，都转换为true 包括包装对象

对象-> 字符串和数值，javascript有两个方法执行转换toString 和 valueOf, 与上面的ToString和ToNumber不同，这两方法直接被暴露出来，除了null的其他所有对象值都有toString方法

toString方法为Object.prototype上的toString方法，调用对象上的[[class]]内部属性

很多内置构造函数原型上，定义了自己的toString方法，返回反映对象的字符串，比如：

  • 数组的 toString 方法将每个数组元素转换成一个字符串，并在元素之间添加逗号后合并成结果字符串
  • 函数的 toString 方法返回源代码字符串
  • 日期的 toString 方法返回一个可读的日期和时间字符串
  • RegExp 的 toString 方法返回一个表示正则表达式直接量的字符串

valueOf方法，表示对象的原始值，默认valueOf方法返回对象本身，日期是一个例外，返回毫秒数

对象转换会调用ToPrimitive(input[， PerferredType])转换为一个基础类型，再调用上面的ToString或ToNumber方法按上面对应转换

  • PerferredType不传值，只有当input为日期类型时，PerferredType为String. 其它情况一律传入Numer，input为原始值直接返回

  • ToPrimitive(obj, Number)，处理如下

    1. 调用valueOf方法，如返回原始值，则返回该原始值，否则下步

    2. 调用toString方法，如果返回原始值，则返回该原始值，否则抛出错误异常

  • ToPrimitive(obj, String)，处理如下

    1. 调用toString方法，如返回原始值，则返回该原始值，否则下步

    2. 调用valueOf方法，如返回原始值，则返回该原始值，否则抛出错误异常

  • 最后调用ToString或ToNumber方法

JSON.stringify

将JavaScript对象转换为一个JSON字符串，实现上也是调用了toString方法，需要注意的点如下

  1. 包装对象在序列化过程中会自动转换为对应的原始值

  2. 传入原始值相当于ToString方法，除了undefined不会被转化为字符串

  3. undefined、函数以及symbol 会被忽略(非数组对象中) 或转化为null(数组对象中)

  4. 如果被序列化的对象有toJSON方法，那么toJSON方法会覆盖原有的默认序列化行为，toJSON的返回值被序列化

  5. 第二个参数replacer，为数组或函数，筛选对象序列化中的属性

隐式类型转化

一元操作符 +

基础类型ToNumber方法对应上面，对象调用ToPrimitive(input, Number)再调用ToNumber方法，如+{} +[‘1’, ‘2’]都为NaN

二元操作符+

当计算value1 + value2时:

  1. lprim = ToPrimitive(value1)
  2. rprim = ToPrimitive(value2)
  3. 如果 lprim 是字符串或者 rprim 是字符串，那么返回 ToString(lprim) 和 ToString(rprim)的拼接结果
  4. 返回 ToNumber(lprim) 和 ToNumber(rprim)的运算结果

注意value为Date对象的时候，PerferredType为String，其余情况Number

1+’2’ -> ‘12’ ， undefined + 1 -> NaN ， undefined + ‘1’ -> ‘undefined1’ ， null + 1 -> 1
